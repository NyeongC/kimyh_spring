스프링 빈을 등록하는 2가지 방법
1. 컴포넌트 스캔과 자동 의존관계 설정
2. 자바 코드로 직접 스프링 빈 등록하기

@SpringBootApplication

같은 패키지 안에서만 자동으로 컴포넌트 스캔을 함

@컴포넌트 싱글톤으로 만들어짐

컨트롤러, 서비스, 리포지토리 등에 @컴포넌트가 들어있음

/*
참고: 스프링은 스프링 컨테이너에 스프링 빈을 등록할 때, 기본으로 싱글톤으로 등록한다(유일하게 하나만
등록해서 공유한다) 따라서 같은 스프링 빈이면 모두 같은 인스턴스다. 설정으로 싱글톤이 아니게 설정할 수
있지만, 특별한 경우를 제외하면 대부분 싱글톤을 사용한다
*/

생성자에 @Autowired 를 사용하면 객체 생성 시점에 스프링 컨테이너에서 해당 스프링 빈을 찾아서
주입한다. 생성자가 1개만 있으면 @Autowired 는 생략할 수 있다

memoryMemeberRepository가 구현체인데 여기에 @리포지토리 설정


DI
생성자 - 한번만 쓰고 변하지도않고 권장
필드 변수 - 뭘 하기 애매
세터 - 중간에 변경 가능함 

스프링 컨테이너에 올라가는 것들만 오토와이어드가 먹음

스프링의 DI를 사용하면 기존 코드를 전혀 손대지않고, 설정을 변경하는것 만으로 구현 클래스를 변경할수 있다.
EX) SpringConfig에서 멤버리포지토리를 구현하는 구현체가 메모리멤버에서 -> jdbc멤버로 변경됨, 설정만 변경으로
